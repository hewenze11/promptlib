# PromptLib 版本更新记录

本文档记录 PromptLib 每个版本的详细改动。所有版本均遵循 [语义化版本](https://semver.org/lang/zh-CN/) 规范。

---

## v1.0.0 (2026-04-09) — 首个正式版本

### 产品背景

人与 AI 的交互日益频繁，但每次开启新对话时，用户都需要重新向 AI 介绍自己的背景、角色设定、项目上下文等高密度信息，效率极低。

PromptLib 的核心思路是：**将这些反复使用的信息片段以「词条」形式管理，在撰写提示词时通过 `@` 引用，点击生成后自动将词条注释附加到文本开头**，让每一次 AI 对话都能快速携带完整上下文。

### 核心功能

#### 词库管理
- 创建词条：每个词条由「名称」（用于 @ 引用）和「详细注释」（完整内容）组成
- 编辑、删除词条
- 词条数量无限制，按创建时间排列

#### 富文本编辑器
- 基于 TipTap 编辑器构建，支持 `@` 触发词条引用弹窗
- 输入 `@` 后实时搜索词库，键盘上下键选择，Enter 确认插入
- 词条插入后以「芯片（chip）」形式内联显示

#### 双模式引用
- **A 模式（高亮紫色 ◆）**：生成文本时，该词条的完整注释会附加在文本开头
- **B 模式（灰色 ◇）**：生成文本时，只输出 `@词条名`，不附加注释
- 点击词条芯片可在 A/B 模式之间切换

#### 文本生成
- 点击「生成文本」，输出格式为：
  ```
  ---词条注释---
  【词条A】详细注释内容
  【词条B】详细注释内容
  --------------
  
  正文内容（@引用处替换为词条名）
  ```
- 支持一键复制生成结果

#### 词库导入导出
- **导出**：将词库下载为 JSON 文件（`promptlib_时间戳.json`）
- **导入**：支持选择文件或拖拽，可选择「合并」或「替换」现有词库
- 词库仅存储在浏览器 localStorage，不上传任何服务器

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | React 18 + Vite |
| 样式 | TailwindCSS v4 |
| 富文本编辑器 | TipTap (StarterKit + Mention extension) |
| 图标 | Lucide React |
| 唯一ID | uuid v4 |
| 容器化 | Docker (nginx:alpine 静态托管) |

### 数据格式

词库文件（JSON）格式如下：

```json
{
  "version": 1,
  "exportedAt": "2026-04-09T00:00:00.000Z",
  "entries": [
    {
      "id": "uuid-v4",
      "title": "词条名称",
      "description": "详细注释内容",
      "createdAt": 1712620800000
    }
  ]
}
```
