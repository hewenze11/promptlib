# PromptLib 版本更新记录

本文档记录 PromptLib 每个版本的详细改动。所有版本均遵循 [语义化版本](https://semver.org/lang/zh-CN/) 规范。

---

## v1.2.0 (2026-04-09) — 词条颜色系统 & 快捷面板

### 新功能

#### 编辑器词条快捷面板
- 编辑器下方新增词条标签云，展示词库中所有词条
- 点击词条标签可直接插入到编辑器光标位置（A 模式），无需手动 @
- 标签颜色实时跟随当前着色模式

#### 词条颜色系统
- 每个词条新增 `color` 字段，颜色数据随词库一起导入导出
- 词库管理编辑词条时可选择颜色：10 种预设色 + 任意自定义颜色（原生颜色选择器）
- 词条列表卡片左侧显示颜色圆点

#### 三种着色模式（UI 层，不影响词库存储）
- **模式一（统一颜色）**：所有词条统一使用默认紫色
- **模式二（层级分色）**：
  - 一级词条（注释中无 @ 引用）→ 天蓝色
  - 二级词条（最高引用一级词条）→ 翡翠绿
  - 三级及以上 → 琥珀色
- **模式三（词库色系）**：使用词条自身存储的 `color` 字段颜色
- 三种模式通过编辑器右上角「Palette」按钮切换，切换不修改词库数据

#### mention 芯片颜色联动
- 通过 @ 触发或点击面板插入时，芯片颜色与当前着色模式同步
- A 模式芯片使用词条颜色渲染，B 模式固定灰色

## v1.1.0 (2026-04-09) — 词条引用增强 & JSON 格式支持

### 新功能

#### 词条注释中支持引用其他词条
- 在编辑词条的「详细注释」中，可以使用 `@其他词条名` 引用已有词条
- 编辑表单实时预览当前注释中引用了哪些词条及其内容
- 生成文本时，被引用词条的注释会**递归展开**：若 A 引用了 B，生成 A 的注释时 B 的内容也会自动填入
- 内置循环引用保护：A→B→A 不会导致死循环，输出 `[循环引用: 词条名]` 提示
- 词条列表中显示该词条引用了哪些其他词条（蓝紫色标签）

#### 词条注释支持 JSON 格式
- 新建/编辑词条时，注释区右上角可切换「文本」/「JSON」两种输入模式
- JSON 模式：等宽字体输入框，实时语法校验，有误时红色提示并禁用保存
- 切换至 JSON 时，若当前内容不是合法 JSON，自动包装为 `{"content": "..."}`
- 词条列表中 JSON 类型词条显示「JSON」徽标

#### 生成文本格式优化
- 正文中词条引用由 `@词条名` 改为 `[词条名]`，更适合 AI 提示词场景，减少歧义

## v1.0.0 (2026-04-09) — 首个正式版本

### 产品背景

人与 AI 的交互日益频繁，但每次开启新对话时，用户都需要重新向 AI 介绍自己的背景、角色设定、项目上下文等高密度信息，效率极低。

PromptLib 的核心思路是：**将这些反复使用的信息片段以「词条」形式管理，在撰写提示词时通过 `@` 引用，点击生成后自动将词条注释附加到文本开头**，让每一次 AI 对话都能快速携带完整上下文。

### 核心功能

#### 词库管理
- 创建词条：每个词条由「名称」（用于 @ 引用）和「详细注释」（完整内容）组成
- 编辑、删除词条
- 词条数量无限制，按创建时间排列

#### 富文本编辑器
- 基于 TipTap 编辑器构建，支持 `@` 触发词条引用弹窗
- 输入 `@` 后实时搜索词库，键盘上下键选择，Enter 确认插入
- 词条插入后以「芯片（chip）」形式内联显示

#### 双模式引用
- **A 模式（高亮紫色 ◆）**：生成文本时，该词条的完整注释会附加在文本开头
- **B 模式（灰色 ◇）**：生成文本时，只输出 `@词条名`，不附加注释
- 点击词条芯片可在 A/B 模式之间切换

#### 文本生成
- 点击「生成文本」，输出格式为：
  ```
  ---词条注释---
  【词条A】详细注释内容
  【词条B】详细注释内容
  --------------
  
  正文内容（@引用处替换为词条名）
  ```
- 支持一键复制生成结果

#### 词库导入导出
- **导出**：将词库下载为 JSON 文件（`promptlib_时间戳.json`）
- **导入**：支持选择文件或拖拽，可选择「合并」或「替换」现有词库
- 词库仅存储在浏览器 localStorage，不上传任何服务器

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | React 18 + Vite |
| 样式 | TailwindCSS v4 |
| 富文本编辑器 | TipTap (StarterKit + Mention extension) |
| 图标 | Lucide React |
| 唯一ID | uuid v4 |
| 容器化 | Docker (nginx:alpine 静态托管) |

### 数据格式

词库文件（JSON）格式如下：

```json
{
  "version": 1,
  "exportedAt": "2026-04-09T00:00:00.000Z",
  "entries": [
    {
      "id": "uuid-v4",
      "title": "词条名称",
      "description": "详细注释内容",
      "createdAt": 1712620800000
    }
  ]
}
```
